## Scope
Асинхронная установка node_exporter на целевой Linux-сервер через Ansible (**по SSH-паролю или по приватному SSH-ключу** — [ADR-067](../../adr/ADR-067-server-ssh-key-auth.md)) и регистрация file_sd-таргета. Детали — [09-provisioning.md](../../09-provisioning.md). Решения — [ADR-004](../../adr/ADR-004-file-sd-registraciya-targetov.md), [ADR-006](../../adr/ADR-006-async-provisioning-bez-brokera.md), [ADR-067](../../adr/ADR-067-server-ssh-key-auth.md).

## Backend — ТЗ
1. Сервис `provision_server(id)`: `status=installing` → decrypt креда → запуск ansible-runner в thread-executor → `online`/`error`. **Ветка выбирается ТОЛЬКО по `servers.auth_method`**, а не по «что не `NULL`» (согласованность материала гарантирует CHECK `ck_servers_auth_material`).
2. **Password-ветка:** креды в inventory/`extravars` **в памяти** (`ansible_user`, `ansible_password`, `target_ip`, `exporter_port`); без записи пароля на диск; `no_log`.
2а. **Key-ветка** (нормативно, [ADR-067](../../adr/ADR-067-server-ssh-key-auth.md) §5, [09-provisioning.md](../../09-provisioning.md#ветка-auth_methodkey-нормативно-adr-067-5)): decrypt ключа (+ фразы) в памяти → **снять парольную фразу в памяти** (пере-сериализация в незашифрованный OpenSSH-PEM; фраза дальше не идёт **никуда**) → записать файл `os.open(..., O_CREAT|O_EXCL|O_WRONLY, 0o600)` внутри `private_data_dir` (`mkdtemp` → `0700`) → `ansible_ssh_private_key_file` в host_vars; **`ansible_password` не задавать вовсе** → `finally`: `os.remove` файла **до** `rmtree` каталога. **Штатный `ansible_runner.run(ssh_key=…)` использовать ЗАПРЕЩЕНО** — на зашифрованном ключе он виснет на приглашении `ssh-add` до `job_timeout` (добавление сервера → ложный `provisioning timeout`).
2б. Ключ расшифровался, но не грузится `cryptography` → `status=error`, `error_message="SSH key unusable"` (**отдельно** от `"SSH connection failed"`: иначе оператор чинит сеть вместо кредов).
3. Таймаут `ANSIBLE_TIMEOUT_SEC` → `error`.
4. Запись `targets/<id>.json` атомарно (temp + `os.replace`) в `FILE_SD_DIR`; формат — [09-provisioning.md](../../09-provisioning.md#регистрация-таргета-file_sd).
5. Удаление: удалить `targets/<id>.json`.
6. Регенерация file_sd из БД при старте (устойчивость к потере volume).
7. Ошибки — человекочитаемые, без секретов; полный вывод — в structlog с маскированием.

## DevOps — ТЗ
1. Плейбук `ansible/install_node_exporter.yml` — идемпотентный (NFR-6), шаги — [09-provisioning.md](../../09-provisioning.md#плейбук-install_node_exporter).
2. node_exporter фиксированной версии ([02-tech-stack.md](../../02-tech-stack.md)), systemd unit, порт `EXPORTER_PORT`.
3. Backend-образ содержит `ansible-core` + `openssh-client` + **`sshpass`** (последний ОБЯЗАТЕЛЕН для password-SSH; без него провижининг падает с `"you must install the sshpass program"`). Требование **сохраняется и после [ADR-067](../../adr/ADR-067-server-ssh-key-auth.md)** — key-режим добавлен **рядом** с паролем, а не вместо. См. [07-deployment.md](../../07-deployment.md#backend-образ).
3а. **Выделенный каталог `ANSIBLE_PRIVATE_DATA_ROOT`** (env, default `/var/run/crm/ansible`) — корень для `private_data_dir` с **расшифрованным** приватным ключом ([ADR-067](../../adr/ADR-067-server-ssh-key-auth.md) §5). Создаётся **в Dockerfile** (`mkdir -p` + `chown app:app` + `0700` — по образцу `FILE_SD_DIR`, образ работает под `USER app`) и перекрывается в проде **отдельным `tmpfs`-mount'ом**, чтобы ключ не касался постоянного диска/слоёв образа/бэкапов.
   - **⛔ `mode` tmpfs — `0o1777`, НЕ `1700`, НЕ голое `1777` и НЕ значение в кавычках:** Docker монтирует tmpfs `root:root`, и если в others нет `w`, `tempfile.mkdtemp` под `app` падает с `PermissionError` — **ломается ВЕСЬ провижининг, включая парольную ветку**. Поле числовое (`uint32`): значение в кавычках отвергается на разборе (падает `docker compose config`/`up` — стек не поднимается вовсе), а голое `mode: 1777` разбирается **десятично** ⇒ `0o3361` (docker 29.6.1 / compose v5.1.4: `1777` → `3361 d-wxrwS--t`, `0o1777` → `1777 drwxrwxrwt`) — тот же отказ прав. Ключ защищают права вложенного `mkdtemp`-каталога (`0700`) и файла (`0600`), а не режим корня.
   - **⛔ `tmpfs` на `/tmp` не монтировать** — туда Starlette спуливает `UploadFile` > ~1 МБ (изображения документов до 5 МБ, [ADR-068](../../adr/ADR-068-documents-image-attachments.md)); общий маленький раздел дал бы `No space left on device`. Каталоги разведены.
   - Сниппеты Dockerfile/compose и обоснование размера — [07-deployment.md](../../07-deployment.md#tmpfs-для-приватных-данных-ansible-нормативно-adr-067-5).
4. `ANSIBLE_HOST_KEY_CHECKING=false` на Этапе 1 ([TD-007](../../100-known-tech-debt.md)).
5. **Открытие firewall на цели (шаг 6 плейбука):** если extravar `scrape_source_ip` задан — идемпотентно открыть `exporter_port` ТОЛЬКО для этого IP; поддержать **ufw** (активен) и **firewalld** (активен); firewall не установлен/не активен → graceful skip без ошибки; НИКОГДА не открывать миру. Если `scrape_source_ip` пуст — задача пропускается. Контракт — [09-provisioning.md](../../09-provisioning.md#шаг-6--открытие-firewall-на-цели-нормативно-реализует-devops).
6. Smoke-тест идемпотентности — [06-testing-strategy.md](../../06-testing-strategy.md).
