# ADR-007 · Шифрование SSH-паролей через Fernet

- Статус: accepted
- Дата: 2026-06-28
- **Амендирован:** [ADR-040](ADR-040-backend-relations-secrets-reverse-lookup.md) — тем же примитивом Fernet/`FERNET_KEY` шифруются **два новых секрета бэка** `api_key`/`admin_api_key` (`backends.api_key_encrypted`/`admin_api_key_encrypted`), по образцу SSH-пароля/пароля прокси/AI-ключа.
- **Амендирован:** [ADR-067](ADR-067-server-ssh-key-auth.md) §1 (2026-07-22) — тем же примитивом и тем же `FERNET_KEY` шифруется **новый класс материала сервера**: приватный SSH-ключ и парольная фраза (`servers.ssh_private_key_encrypted`/`ssh_key_passphrase_encrypted bytea`). Перечень секретов в §Решении (`ssh_password_encrypted`) перестал быть исчерпывающим; сам примитив, схема ключа и остаточные риски — **без изменений**.

## Контекст

SSH-пароли целевых серверов нужны Ansible для провижининга и должны храниться в БД зашифрованными at-rest (ТЗ, NFR-2). Требуется обратимое шифрование (пароль нужно расшифровать перед использованием), не хэш.

## Решение

Шифровать SSH-пароли симметрично через **Fernet** (`cryptography`): AES-128-CBC + HMAC-SHA256, со встроенной аутентификацией и временной меткой. Ключ `FERNET_KEY` (base64, 32 байта) — из `.env`, вне БД и репозитория. В БД — `ssh_password_encrypted bytea`. Расшифровка только в памяти провижининг-сервиса перед запуском Ansible; значение не логируется и не возвращается API.

## Обоснование

- Обратимость обязательна (хэш не подходит — пароль нужен в открытом виде для SSH).
- Fernet — высокоуровневый, безопасный по умолчанию примитив (не нужно вручную выбирать режим/IV), есть в стандартной экосистеме Python.
- Ключ вне БД → компрометация дампа БД не раскрывает пароли.

## Последствия

- (+) Простое, надёжное шифрование «из коробки».
- (+) Аутентификация ciphertext (защита от подмены).
- (−) Симметричный ключ — единая точка: утечка `FERNET_KEY` + дамп БД раскрывает пароли. Митигация: строгий контроль секрета, ротация через `MultiFernet` ([TD-006](../100-known-tech-debt.md)).
- (−) Нужно безопасно управлять `FERNET_KEY` и его ротацией.

## Альтернативы

- **Хэширование (bcrypt/argon2)** — неприменимо: нужен обратимый секрет.
- **Внешний secret manager (Vault)** — отложено: избыточно для Этапа 1, но совместимо в будущем (хранить там `FERNET_KEY`).
- **SSH-ключи вместо паролей** — вне ТЗ Этапа 1 (модалка запрашивает пароль); рассмотреть в будущем как более безопасный вариант.
