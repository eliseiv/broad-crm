# ADR-023 · UI-пакет: вертикальный дропдаун-навигации + «квадратная» панель, карточка прокси — только IP, единственная кнопка «Удалить» на карточке

- Дата: 2026-07-08

## Контекст

Пакет из трёх UI-правок по прямому запросу пользователя (правки 1, 2а, 3а батча). Все три — изменения **дизайн-системы/отображения**, без изменения API-контрактов, схем БД и бизнес-логики:

1. **Навигация (категории-дропдауны, [ADR-022](ADR-022-teams-nav-categories.md)).** Панель категории-дропдауна (Агрегатор / Мониторинг / Пользователи) в текущей реализации визуально раскрывает пункты неудобно (в один ряд/с большим скруглением). Требование: пункты-страницы располагать **вертикально — по одному в строку**, а саму панель сделать **«квадратнее»** — уменьшить радиус скругления углов.
2. **Карточка прокси.** Сейчас `ProxyCard` показывает адрес `host:port`, а также логин (`username`) и индикатор наличия пароля (`has_password`). Требование: на карточке отображать **только IP-адрес** прокси (`host`); Логин, Пароль и Порт **не показывать**.
3. **Карточка бэка (страница «Бэки»).** По факту на карточке недоступного бэка появляется **вторая** кнопка «Удалить» (в блоке error-состояния, в дополнение к основной). Требование: оставить **одну** кнопку «Удалить» (верхнюю/основную), убрать нижнюю (появляющуюся только при недоступности).

## Решение

1. **Дропдаун-навигация — вертикальная колонка пунктов + «квадратная» панель.**
   - Пункты категории (`NavLink`) в панели `NavMenu` располагаются **вертикально, один пункт на строку** (стек/колонка), а не в горизонтальный ряд. Каждый пункт — полноширинная строка панели (кликабельная область на всю ширину), активный/hover — подсветка по токенам (`--accent` / `--text-primary`).
   - Панель дропдауна — **менее скруглённая** («квадратнее»): радиус скругления панели — **6px** (нормативно), вместо более круглых значений карточек/кнопок (16/12/8–10px). Явный дизайн-токен — см. [08-design-system.md](../08-design-system.md#сетка-отступы-скругления).
   - Прочее поведение `NavMenu` (клавиатурная доступность, Esc, стрелки, возврат фокуса, `chevron-down`) и видимость категории (по доступности хотя бы одного пункта) — без изменений ([ADR-022](ADR-022-teams-nav-categories.md)).

2. **`ProxyCard` — только IP.** Карточка прокси отображает из адресных полей **только `host`** (IP/FQDN прокси) — без порта (`port`), без логина (`username`) и без индикатора пароля (`has_password`). Имя (`name`), тип (`proxy_type`), статус-бейдж, причина ошибки и кнопка «Удалить» — остаются. Это **правка отображения**: API-контракт `ProxyListItem` **не меняется** (`port`/`username`/`has_password` по-прежнему присутствуют в ответе и используются формой редактирования и монитором), меняется только набор полей, рендерящихся на карточке. Telegram-алерты прокси (идентификация `<host>:<port>`) — без изменений.

3. **Единственная кнопка «Удалить» на карточке.** На карточке (бэка — и, единообразно, любой ресурсной карточки со статусом) кнопка «Удалить» — **ровно одна** (основная, в блоке действий карточки). Отдельная кнопка «Удалить», ранее дублировавшаяся в error-состоянии (`check_status='error'` / недоступность), **убирается**: удаление доступно в любом статусе через единственную кнопку. Причина ошибки при `error` по-прежнему показывается строкой, но без второй кнопки.

## Обоснование

- Вертикальная раскладка пунктов — стандартный паттерн выпадающего меню (список действий сверху вниз), читается однозначно при 3–4 пунктах в категории «Мониторинг»/«Пользователи».
- «Квадратная» панель (6px) визуально отделяет всплывающее меню от скруглённых карточек-поверхностей, соответствует enterprise-эстетике (Linear/Vercel — дропдауны заметно менее скруглены, чем карточки).
- Прокси-карточка «только IP» убирает визуальный шум (логин/порт/пароль редко нужны при беглом просмотре; полный набор доступен в форме редактирования). Секрет и так не показывался; логин/порт — детали, вынесенные в edit.
- Единственная кнопка «Удалить» устраняет дубль-контрол: две кнопки с одним действием — источник путаницы и лишний тап; удаление не должно зависеть от статуса.

## Последствия

- (+) Чище шапка навигации и карточки; нет дубль-кнопок; отображение прокси лаконичнее.
- (+) Контракты API и модель БД не затронуты — правки чисто фронтовые (дизайн-система).
- (−) На карточке прокси не видно порт/логин без открытия edit — осознанный компромисс (беглый просмотр по IP + статусу).
- (−) Удаление недоступного бэка теперь только через основную кнопку (не дублируется рядом с причиной ошибки) — эргономически эквивалентно.

## Альтернативы

- **Оставить панель со скруглением карточек** — отвергнуто: пользователь явно просит «квадратнее».
- **Показывать `host:port` на карточке прокси** — отвергнуто: запрос — «только IP».
- **Оставить дубль-кнопку «Удалить» в error-состоянии** — отвергнуто: две кнопки одного действия избыточны; удаление не должно быть привязано к статусу.
