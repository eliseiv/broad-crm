# ADR-029 · UI/UX-пакет: экран «Придумайте пароль», убраны заголовки страниц Users/Roles/Teams, лидер-из-участников, рестайл нав-дропдауна и разделитель категорий

- Статус: accepted
- Дата: 2026-07-08

## Контекст

Правки 1, 4–10 батча — прямые UX-решения пользователя по фидбеку с прода. Контракты API и модели БД **не меняются** (кроме связанных ADR-027/ADR-028) — это правки нормативного дизайн-словаря и UX-описания. ADR фиксирует их, т.к. часть меняет **действующий нормативный текст** ADR-022/023/025/026 (принцип «изменение нормативного текста docs — через architect»), а не только добавляет.

## Решение

### 1. Экран «Придумайте пароль» при первом входе (правка 1, амендмент [ADR-025](ADR-025-passwordless-users-login-identifier-open-first-login.md))

- Заголовок экрана установки пароля первого входа переименован: **«Задайте пароль» → «Придумайте пароль»** (везде в UI-словаре и UX-описании).
- **Беспарольный пользователь не вводит никакого пароля на шаге логина.** Поле «Пароль» шага 2 для него остаётся пустым (оно и так необязательно на клиенте — [ADR-025](ADR-025-passwordless-users-login-identifier-open-first-login.md)); после `POST /api/auth/login` с ответом `password_setup_required: true` показывается экран **«Придумайте пароль»**, где пользователь **сам придумывает и вводит** свой пароль (≥ 8 символов).
- **Без генератора / без случайного пароля.** Экран не предлагает и не требует ввода системно-сгенерированного/случайного пароля — только пользовательский. Одно поле «Новый пароль» (`type=password`, toggle видимости, ≥ 8) + кнопка «Сохранить» → `POST /api/auth/set-password` (Bearer setup-token) → сразу залогинен.
- **Контракт `POST /api/auth/login` / `POST /api/auth/set-password` НЕ меняется** ([04-api.md](../04-api.md#auth), [ADR-025](ADR-025-passwordless-users-login-identifier-open-first-login.md)); правка — заголовок экрана и UX-формулировка потока.

### 2. Убраны заголовки страниц Users / Roles / Teams (правки 4/5/6)

- На страницах `/users`, `/roles`, `/teams` **не рендерятся** H1-заголовок (соответственно «Пользователи» / «Роли» / «Команды») **и** подпись-описание под ним («Учётные записи…» / «Роли и матрица…» / «CRM-команды…»). Страница начинается сразу с контента (список/кнопка действия). Идентификация раздела — через активный пункт навигации, дублирующий H1 на странице избыточен.
- Пункты **навигации** этих разделов (лейблы «Пользователи»/«Роли»/«Команды» в меню) **сохраняются** — убирается только внутристраничный заголовочный блок (H1 + subtitle).

### 3. Форма команды: лидер из участников, первый добавленный = лидер (правки 7/8, UX-уточнение [ADR-026](ADR-026-teams-optional-leader-auto-transfer.md))

- В форме создания/редактирования команды поле **«Лидер»** выбирается **ТОЛЬКО из выбранных Участников** этой команды (не из всех пользователей `GET /api/users`). Пока участники не выбраны — выбор лидера недоступен (пусто).
- **Дефолт лидера = первый добавленный участник** (авто-назначение): при добавлении первого участника он становится лидером; UI отражает это. Соответствует серверному авто-назначению `team_service.create_team` ([ADR-026](ADR-026-teams-optional-leader-auto-transfer.md)).
- **Пустой состав → без лидера** (`leader_id = NULL`) — **единственный** легитимный кейс «Без лидера». Прежнее отображение «Без лидера» у команд с участниками было багом (лидер не подставлялся) — устраняется тем, что первый участник авто-становится лидером.
- **Контракт `/api/teams` НЕ меняется** ([04-api.md](../04-api.md#teams)): backend уже авто-назначает первого участника лидером ([ADR-026](ADR-026-teams-optional-leader-auto-transfer.md)); правка — UX-поведение формы (источник кандидатов в лидеры = выбранные участники, дефолт = первый).

### 4. Рестайл панели нав-дропдауна (правка 9, амендмент [ADR-023](ADR-023-ui-nav-dropdown-proxy-ip-single-delete.md))

Панель выпадающего меню категории (`NavMenu`) перестилизуется:
- **Фон — как у хэдера** (`--bg-base`), а не `--surface-1`.
- **Без видимой границы** (убрана тонкая рамка панели, заданная [ADR-023](ADR-023-ui-nav-dropdown-proxy-ip-single-delete.md)).
- **Уже** — ширина панели по ширине триггера/контента, **без фиксированной min-width** (ранее ~180px).
- **Пункты выровнены по левому краю с названием категории** — убран горизонтальный внутренний паддинг панели, из-за которого пункты были смещены на ~6px правее триггера.
- **Сохраняется** от [ADR-023](ADR-023-ui-nav-dropdown-proxy-ip-single-delete.md): вертикальная раскладка (по одному пункту в строку) и радиус скругления **6px**.

### 5. Разделитель «|» между категориями (правка 10)

- В хэдере между категориями **Агрегатор / Мониторинг / Пользователи** отображается визуальный разделитель **«|»** приглушённым тоном (`--text-secondary` или эквивалент). Разделитель — чисто визуальный (декоративный, `aria-hidden`), между смежными **видимыми** категориями (если категория скрыта по правам — лишний разделитель не рендерится).

## Обоснование

- **Все правки — UX-слой**, контракты/БД не затронуты (за исключением связанных ADR-027/028). Фиксация в ADR нужна, т.к. правки 1/4-6/9 **меняют действующий нормативный текст** ([ADR-025](ADR-025-passwordless-users-login-identifier-open-first-login.md) заголовок, [ADR-023](ADR-023-ui-nav-dropdown-proxy-ip-single-delete.md) стиль панели) — по правилу проекта такие изменения проходят через architect, а тела старых ADR не переписываются (добавляется маркер амендмента).
- **«Придумайте пароль»** точнее «Задайте пароль»: акцент на том, что пользователь **сам придумывает** пароль (не система выдаёт).
- **Убранные H1** — раздел уже обозначен активным пунктом навигации; дублирующий заголовок съедает вертикаль без пользы.
- **Лидер из участников** — прямое устранение бага «везде Без лидера» и согласование UX с серверным авто-назначением ([ADR-026](ADR-026-teams-optional-leader-auto-transfer.md)); выбор лидера из всех пользователей противоречил инварианту «лидер ∈ участники».
- **Рестайл дропдауна** — визуальная интеграция панели с хэдером (единый фон, без рамки, выравнивание) по прямому фидбеку.

## Последствия

- (+) Онбординг-экран называется понятнее; беспарольный пользователь не путается с полем пароля на шаге логина.
- (+) Страницы Users/Roles/Teams компактнее (нет дублирующего H1).
- (+) Форма команды корректно назначает лидера (первый участник), «Без лидера» — только для пустой команды.
- (+) Нав-дропдаун визуально согласован с хэдером.
- (−) Меняется нормативный текст [ADR-023](ADR-023-ui-nav-dropdown-proxy-ip-single-delete.md) (стиль панели) и [ADR-025](ADR-025-passwordless-users-login-identifier-open-first-login.md) (заголовок экрана) — оформлено как амендмент (маркеры в старых ADR, тела не переписаны).
- (−) Визуальные/layout-правки (нав-дропдаун, разделитель, убранные заголовки) требуют проверки фактического рендера (не только сверки классов) — стандартное требование для layout-изменений.

## Альтернативы

- **Оставить «Задайте пароль»** — отвергнуто: пользователь просил «Придумайте пароль».
- **Оставить H1 на страницах** — отвергнуто по прямому запросу (дублирует навигацию).
- **Лидер из всех пользователей (как было)** — отвергнуто: противоречит инварианту «лидер ∈ участники» и порождал баг «Без лидера»; лидер обязан быть участником.
- **Отдельный ADR на каждую UI-правку** — отвергнуто: правки мелкие, тематически связаны (UI/UX-пакет одного батча); единый ADR проще для трассировки.
- **Разделитель как отдельный элемент между всеми слотами (вкл. скрытые)** — отвергнуто: разделитель ставится только между смежными видимыми категориями (иначе висячие «|»).
