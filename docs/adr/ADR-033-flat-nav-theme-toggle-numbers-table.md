# ADR-033 · UI-пакет: плоская навигация + светлая/тёмная тема + упрощение таблицы «Номера»

- **Статус:** accepted
- **Дата:** 2026-07-09
- **Спринт:** C
- **Амендмент к:** [ADR-022](ADR-022-teams-nav-categories.md) (навигация), [ADR-023](ADR-023-ui-nav-dropdown-proxy-ip-single-delete.md)/[ADR-029](ADR-029-ui-login-password-nav-team-form.md) (рестайл нав-дропдауна), [ADR-030](ADR-030-sms-module-full-merge.md) (таблица «Номера»)
- **Связано:** [08-design-system.md](../08-design-system.md), [modules/sms](../modules/sms/README.md), [00-vision.md](../00-vision.md)

## Контекст

Три продуктовых требования по UI/UX CRM-фронтенда (одним пакетом, по образцу UI-пакетов [ADR-023](ADR-023-ui-nav-dropdown-proxy-ip-single-delete.md)/[ADR-029](ADR-029-ui-login-password-nav-team-form.md)):

1. **Плоская навигация.** Страницы (Почты, СМС, Серверы, ИИ-ключи, Прокси, Бэки, Пользователи, Роли, Команды) должны отображаться в хэдере **напрямую, без разделения на категории** «Агрегатор / Мониторинг / Пользователи». Текущая модель ([ADR-022](ADR-022-teams-nav-categories.md)) — 3 категории-дропдауна через компонент `NavMenu` с разделителем «\|» ([ADR-029](ADR-029-ui-login-password-nav-team-form.md)).
2. **Светлая/тёмная тема.** Нужна иконка-переключатель светлого/тёмного фона сервиса. Сейчас существует **только тёмная** тема (все токены в `:root` тёмные; `[data-theme]`/`prefers-color-scheme` для цветов нет).
3. **Упрощение таблицы «Номера».** На странице «Номера» убрать колонку «Действия»; иконку-карандаш инлайн-редактирования показывать **постоянно** (не только по hover).

## Решение

### 1. Плоская навигация (упраздняет категории-дропдауны)

- Навигация в хэдере `AppLayout` — **плоский горизонтальный ряд `NavLink`** без категорий-дропдаунов. Категории-группы «Агрегатор» / «Мониторинг» / «Пользователи» как визуальные триггеры и разделитель «\|» между ними **упразднены**; компонент `NavMenu` для основной навигации больше не используется.
- **Порядок пунктов (нормативно):** Почты (`/mail`), СМС (`/sms`), Серверы (`/servers`), ИИ-ключи (`/ai-keys`), Прокси (`/proxies`), Бэки (`/backends`), Пользователи (`/users`), Роли (`/roles`), Команды (`/teams`). Порядок сохраняет прежнюю логику группировки (агрегатор → мониторинг → пользователи), но без визуальных заголовков-категорий. Совпадает с плоским порядком листьев `DefaultRoute`.
- **Гейтинг видимости сохраняется** (только UX): пункт виден ⇔ `<page>:view ∈ permissions` (или `is_superadmin`); `users` — `is_superadmin || role=="admin"`. Скрытый по правам пункт не рендерится. Граница безопасности — серверный `403`.
- **Порядок резолва дефолт-маршрута (`DefaultRoute.tsx`) НЕ меняется** — остаётся `mail → sms → servers → ai-keys → proxies → backends → users → roles → teams` ([ADR-022](ADR-022-teams-nav-categories.md), [ADR-030](ADR-030-sms-module-full-merge.md)). Меняется исключительно **визуальная группировка** хэдера. «Дашборд» по-прежнему вне меню (по прямому URL под `dashboard:view`).
- **Деградация на узких вьюпортах — горизонтальный скролл** ряда навигации (`overflow-x-auto` + утилита `scrollbar-none`, полоса скрыта, прокрутка сохранена), высота хэдера фиксирована (`flex-nowrap`, без переноса — не ломает `sticky`/full-bleed). Бургер-меню **рассмотрено и отклонено** (лишний stateful-компонент ради узких вьюпортов во внутренней преимущественно-desktop админ-панели; при росте числа пунктов можно ввести позже).

### 2. Светлая/тёмная тема (`data-theme` на `<html>`, дефолт — тёмная)

- **Носитель — атрибут `data-theme="dark|light"` на `document.documentElement`.** Тёмные значения CSS-переменных — на `:root` (и `[data-theme=dark]`), светлые — под `[data-theme=light]`. Компоненты и Tailwind-токены ссылаются на переменные — при смене темы меняются только значения переменных.
- **Дефолт при первом входе — СИСТЕМНАЯ тема ОС** (решение пользователя, [Q-UI-2](../99-open-questions.md)). При отсутствии сохранённого выбора тема следует `prefers-color-scheme` (`matchMedia('(prefers-color-scheme: dark)')` → `dark`, иначе `light`). **Явный выбор пользователя переопределяет систему** (`localStorage['crm-theme']` приоритетнее ОС и «залипает» между сессиями). Пока явного выбора нет — приложение **подписывается на `prefers-color-scheme` change** и следует за сменой темы ОС; после первого явного выбора сохранённое значение имеет приоритет. (Прежний dark-first-дефолт заменён на системный по прямому решению пользователя; бренд остаётся тёмным-по-духу, но стартовая тема уважает настройку ОС.)
- **Персистентность — `localStorage['crm-theme']`** (`"light"`/`"dark"`; пусто = «выбор не сделан» → системный дефолт).
- **Без FOUC — синхронный inline-скрипт в `<head>` `index.html`**: вычисляет тему как **сохранённый выбор → иначе `matchMedia('(prefers-color-scheme: dark)')`** и сразу проставляет `data-theme` до первой отрисовки.
- **`color-scheme`** переключается с темой (`[data-theme=light] { color-scheme: light }`) — нативные контролы следуют теме.
- **Переключатель-иконка в хэдере `AppLayout`:** `Sun` в тёмной (→ светлая, `aria-label` «Светлая тема»), `Moon` в светлой (→ тёмная, «Тёмная тема»); `variant="ghost"`, рядом с «Выйти».
- **Полная светлая палитра** для каждого токена (bg/surface-1/2/3, border-subtle/strong, text-primary/secondary/tertiary, accent/accent-hover, status-green/yellow/red, gauge-track, gradients дуги) + **тема-зависимые тени** (`--shadow-card`/`--shadow-sub` — новые переменные вместо статичных `boxShadow` в конфиге) — таблица в [08-design-system.md · Цветовые токены](../08-design-system.md#цветовые-токены-css-custom-properties--tailwind-theme). Контраст текста ≥ WCAG AA (NFR-8), продуманная светлая тема (белые карточки на светло-сером фоне), не «инверсия в лоб».
- **Исключения:** Mini App `/tg/sms` (вне `AppLayout`) продолжает наследовать **тему Telegram** (`themeParams`) — переключателя нет, `data-theme` на неё не влияет ([ADR-031](ADR-031-sms-operator-mini-app.md) не меняется). Sandbox-iframe тела письма — собственный документ, тема не распространяется. Экран входа (`/login`, вне `AppLayout`) переключателя не имеет — рендерится в сохранённой теме, а при её отсутствии — в системной (inline-скрипт применяет `data-theme` до рендера).

### 3. Упрощение таблицы «Номера»

- **Колонка «ДЕЙСТВИЯ» (`Select` команд + «Перенести» + «Удалить») упразднена.** Функции перераспределены:
  - **Перенос** → колонка **«Команда»**: `Select` коммитит перенос **сразу при выборе** (`useTransferSmsNumber`, toast «Номер перенесён»), без отдельной кнопки «Перенести». Без права `sms:transfer` — имя команды текстом (read-only).
  - **Удаление** → **одиночная иконка-кнопка `Trash2`** (danger) в последней колонке **без текстового заголовка** (не labeled-колонка «Действия»), confirm-модалка не меняется. Гейт `sms:delete`.
- **RBAC/API не меняются:** действия `sms:transfer`/`sms:delete` остаются в каталоге и в API (`POST /api/sms/numbers/{id}/transfer`, `DELETE /api/sms/numbers/{id}`) — меняется только их размещение, не наличие. Альтернатива «убрать удаление совсем» — [Q-UI-3](../99-open-questions.md); принята текущая (удаление сохранено компактной иконкой).
- **Карандаш инлайн-редактирования (`Логин`/`Приложение`/`Примечание`) виден ПОСТОЯННО.** Прежнее `opacity-0 group-hover:opacity-100` в `InlineEditField` заменяется на постоянную видимость при `canEdit`. Меняется дефолт видимости карандаша в общем `InlineEditField`.

## Последствия

- Разметка/классы компонентов при смене темы не трогаются — переключение полностью на уровне CSS-переменных; корректность зависит от того, что **все** цвета берутся через переменные (регрессия: захардкоженный hex в компоненте не переключится — reviewer обязан это проверить).
- `NavMenu` и разделитель «\|» становятся неиспользуемыми для основной навигации; их удаление из кода — за frontend (orphaned-тесты `NavMenu`, если есть — задача qa, не блокер исполнителя).
- Тени переезжают в переменные — `tailwind.config.ts` `boxShadow.card`/`.sub` начинают ссылаться на `var(--shadow-card)`/`var(--shadow-sub)`.
- Продуктовые развилки (дефолт темы система-vs-тёмная; судьба удаления номера) зафиксированы как [Q-UI-2](../99-open-questions.md)/[Q-UI-3](../99-open-questions.md) — приняты дефолтные решения, реализация не блокируется.
- Контракты API и модель БД **не затронуты**.

## Альтернативы

- **Навигация:** бургер-меню на узких вьюпортах (отклонено — избыточно, см. выше); сохранить категории-дропдауны (отклонено — прямое требование продукта на плоский ряд).
- **Тема:** дефолт всегда тёмная, без системного детекта (**отклонено** решением пользователя в пользу **системного дефолта** `prefers-color-scheme`, [Q-UI-2](../99-open-questions.md)); тема через class `dark`/`light` на `<html>` (Tailwind `darkMode:'class'`) вместо `data-theme` — эквивалентно, но `data-theme` не конфликтует с существующим `darkMode:'class'` и явнее выражает трёхзначность; выбран `data-theme`.
- **Номера:** удалить перенос/удаление из UI совсем (отклонено — потеря возможностей; [Q-UI-3](../99-open-questions.md)); оставить «Действия» лейбл-колонку с одной кнопкой (отклонено — не удовлетворяет требованию «убрать поле Действия»).
