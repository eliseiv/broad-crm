# ADR-005 · Кастомные SVG-спидометры vs Grafana embed

- Статус: accepted
- Дата: 2026-06-28

## Контекст

На главной странице карточки серверов содержат спидометры CPU/RAM/SSD. Варианты: встраивать Grafana-панели (iframe/embed) или рисовать собственные SVG-компоненты. ТЗ требует кастомные SVG, точную композицию по референсу и цвет дуги строго по нагрузке (зелёный/жёлтый/красный), а также высокий enterprise-уровень дизайна.

## Решение

На главной странице — **собственные SVG-спидометры** (без Grafana-панелей и chart-библиотек). Данные берутся из Prometheus через backend API (PromQL). Grafana используется **только для drill-down** — детальный анализ по ссылке из карточки.

Спецификация компонента Gauge — [08-design-system.md](../08-design-system.md#компонент-gauge-кастомный-svg).

## Обоснование

- Полный контроль над внешним видом (форма дуги 270°, градиент по зоне, свечение, моноцифры) — невозможен с embed.
- Цвет строго по зоне нагрузки — собственная логика, единая с backend.
- Нет тяжёлых iframe, лучше производительность и a11y (`role="meter"`).
- Grafana всё равно нужна для истории/детализации — это её сильная сторона.

## Последствия

- (+) Точное соответствие референсу и бренду, лёгкие компоненты.
- (+) Единые пороги зон frontend↔backend.
- (−) Нужно самим реализовать SVG-математику дуги и анимацию (специфицировано в дизайн-системе).
- (−) Дублирование визуализации (gauge на главной + графики в Grafana) — осознанно.

## Альтернативы

- **Grafana embed/iframe** — отвергнуто: нет контроля над дизайном, тяжело, не отвечает цвету-по-нагрузке и референсу.
- **Chart-библиотека (Recharts/ECharts)** — отвергнуто: избыточно для простого gauge, хуже контроль над формой/свечением.

## Поправка (2026-06-30) — удаление drill-down ссылки из карточки

Ссылка drill-down из карточки сервера на Grafana **удалена** из UI Этапа 1 (вместе с мёртвой конфигурацией `VITE_GRAFANA_URL` и `GRAFANA_BASE_URL`).

- **Центральное решение ADR-005 не меняется:** на главной — кастомные SVG-спидометры, без Grafana-embed/iframe.
- **Grafana остаётся** в составе compose как datasource-only ([TD-010](../100-known-tech-debt.md)) и доступна **напрямую** через proxy `/grafana` (Explore); из карточки на неё больше нет ссылки.
- Затронуто: [modules/ui](../modules/ui/README.md), [01-architecture.md](../01-architecture.md), [00-vision.md](../00-vision.md), [05-security.md](../05-security.md), [07-deployment.md](../07-deployment.md#переменные-окружения).
- Статус ADR-005 — по-прежнему `accepted` (поправка сужает UI-scope, не отменяет решение о gauge).
