# ADR-036 · Фильтр «Все команды» на `/sms` — только для admin-уровня (+ `sees_all_sms_teams` в `GET /api/auth/me`)

- Статус: accepted
- Дата: 2026-07-09
- Амендмент: [ADR-032](ADR-032-sms-visibility-admin-full-catalog.md) (видимость SMS по полному каталогу) — UI-следствие

## Контекст

На вкладке «Сообщения» страницы `/sms` два фильтра-дропдауна: **«Все номера»** и **«Все команды»**. По [ADR-032](ADR-032-sms-visibility-admin-full-catalog.md) видимость SMS/номеров ограничена scope: актор **admin-уровня** (`sees_all_teams = is_superadmin OR роль владеет полным каталогом прав`) видит **все** команды; **прочие роли** видят только SMS/номера **своих** команд (по `user_teams`).

Для не-admin пользователя фильтр «Все команды» **бесполезен и вводит в заблуждение**: он и так видит только свои команды (scope сужает результат независимо от выбора), а сам дропдаун рендерится пустым/неактивным («поле есть, но выбрать ничего нельзя»). Требование пользователя: у не-admin роль фильтр «Все команды» не должен показываться вовсе.

**Проблема источника истины:** `sees_all_teams` вычисляется на backend (`deps.get_sms_scope`, [ADR-032](ADR-032-sms-visibility-admin-full-catalog.md)) и фронту напрямую не отдаётся. `GET /api/auth/me` отдаёт `is_superadmin` и `permissions`, но **не** предикат «полный каталог». Фронт не может надёжно вычислить его сам: для `permissions_subset(full_catalog, perms)` нужен полный каталог из `GET /api/permissions/catalog`, который гейтится `roles:view` — не у всех операторов есть это право → фронт-вычисление хрупко и дублирует backend-логику.

## Решение

### 1. `GET /api/auth/me` отдаёт производный булев признак `sees_all_sms_teams`

- Поле `sees_all_sms_teams: boolean` = backend-предикат `is_superadmin OR permissions_subset(full_catalog_permissions(), permissions)` — **тот же**, что вычисляет `get_sms_scope` для SMS-scope ([ADR-032](ADR-032-sms-visibility-admin-full-catalog.md)). Вычисляется в handler'е `/api/auth/me` из `Principal` (нулевая стоимость — `permissions` уже загружены).
- **Backend — единственный источник истины** (вариант A). Фронт **не** повторяет `permissions_subset`/не тянет полный каталог — берёт готовый флаг. Работает для всех пользователей (auth/me доступен каждому аутентифицированному, не требует `roles:view`).
- Отклонён **вариант B** (фронт вычисляет полный каталог из `permissions`): дублирование доменной логики + зависимость от `GET /api/permissions/catalog` (под `roles:view`, не у всех есть).

### 2. UI-правило показа фильтра «Все команды» (нормативно)

- На вкладке «Сообщения» `/sms` фильтр **«Все команды»** рендерится **только** при `sees_all_sms_teams === true`. Для прочих ролей он **не рендерится** вовсе (не пустой/не disabled — отсутствует).
- Фильтр **«Все номера»** рендерится **всем** держателям `sms:view` (у не-admin — только его номера по scope).
- Это UI-следствие [ADR-032](ADR-032-sms-visibility-admin-full-catalog.md): для не-admin scope и так сужает выборку до своих команд — отдельный team-фильтр избыточен.

### 3. Безопасность

Правило — **UX**, не граница безопасности: серверный SMS-scope ([ADR-032](ADR-032-sms-visibility-admin-full-catalog.md)) уже гарантирует, что не-admin получает только SMS своих команд **независимо** от переданных `team_id` (вне scope → пустой результат, анти-энумерация). Скрытие фильтра лишь убирает бесполезный контрол. `sees_all_sms_teams` — производный небезопасный-к-раскрытию булев признак (не раскрывает состав прав сверх уже отдаваемых `permissions`/`is_superadmin`).

## Последствия

- `docs/04-api.md`: `GET /api/auth/me` += `sees_all_sms_teams: boolean` (производное).
- `docs/05-security.md`: раздел видимости SMS ([ADR-032](ADR-032-sms-visibility-admin-full-catalog.md)) — отметка, что предикат отдаётся фронту через auth/me как `sees_all_sms_teams` (UI-гейт фильтра).
- `docs/08-design-system.md`: §Вкладка «Сообщения» — фильтр «Все команды» под `sees_all_sms_teams`.
- Backend: handler `/api/auth/me` добавляет поле; схема `MeResponse += sees_all_sms_teams`. Frontend: `useAuthMe`/auth-store типизирует поле; тулбар «Сообщения» условно рендерит team-фильтр.

## Альтернативы (отклонены)

- **Вариант B (фронт вычисляет полный каталог)** — дублирование `permissions_subset`, зависимость от каталога под `roles:view`.
- **Оставить фильтр пустым/disabled** — текущее поведение, признано багом (мусорный контрол).
- **Новое право `sms:view_all`** — [ADR-032](ADR-032-sms-visibility-admin-full-catalog.md) уже отклонил его; переиспользуется предикат полного каталога.
